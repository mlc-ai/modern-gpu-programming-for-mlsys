MBarrier：数据结构与 API

MBarrier Object（64-bit，位于 shared memory）

phase
0 或 1
pending_count
剩余 arrival
expected_count
预期总数
tx_count
待完成字节数
phase 当前 phase（0 或 1）。完成后会自动翻转。
pending_count 当前 phase 完成前还需要多少次 arrival。
expected_count 每个 phase 预期的 arrival 总数（由 init 设置）。
tx_count 尚未完成的异步传输字节数。arrive.expect_tx 会增加它；传输完成后硬件会减少它。

Phase 完成条件

pending_count == 0
&&
tx_count ≤ 0
当 所有预期 thread 都已 arrive，并且 所有预期异步字节都已传输完成 时，一个 phase 才算完成。 完成后，barrier 会自动 reset：phase 翻转，pending_count 重置为 expected_count，tx_count 重置为 0。

核心 API

Instruction角色对 MBarrier 的影响
mbarrier.init setup 用 expected arrival count 初始化 barrier phase=0, pending=count, tx=0
mbarrier.arrive producer Thread 向 barrier 发出 arrival 信号 pending_count -= 1
mbarrier.arrive.expect_tx producer Arrive，同时声明预期的异步传输字节数 pending -= 1, tx_count += txCount
tcgen05.commit producer Tensor Core 发出 MMA 完成信号 arrive::one on mbarrier
mbarrier.try_wait consumer 等待 phase 完成，期间挂起 thread 阻塞直到 phase done
三种 arrival 模式： (1) mbarrier.arrive — thread 直接减少 pending_count。 (2) arrive.expect_tx — thread arrive 并声明字节数；TMA engine 在传输完成后自动减少 tx_count。 (3) tcgen05.commit — MMA 完成时 Tensor Core 自动 arrive。
